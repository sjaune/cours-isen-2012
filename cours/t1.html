HTML

Définition

HTML : Hypertext Markup Language

  • HTML est un langage hypertexte (hyperliens dans du texte)
  • C'est un langage à balises (tags)
  • Les balises servent à décrire le contenu
  • Document HTML = Page web

Notions de base

Notions de base

Les balises

  • Les balises HTML sont des mots-clés entourés d'équerres (ex. <html>)
  • Les balises HTML vont normalement par paire (ex. <b> et </b>)
  • La première balise est celle d'ouverture et la deuxième celle de fermeture
  • La balise de fermeture s'écrit comme une balise d'ouverture mais avec une barre oblique (slash) devant (ex. </body>)
  • Un élement HTML est constitué de ce qui se trouve entre deux balises : 
    <balise>Contenu</balise>
  • Le rôle d'un navigateur Internet (Chrome, Internet Explorer, Firefox, ...) est de lire le code des documents HTML et de les afficher en tant que page web

Notions de base

Les balises

L'imbrication des balises est possible :

<div>
    <ul>
        <li>Texte 1</li>
        <li>Texte 2</li>
        <li>Texte 3</li>
    </ul>
</div>
  • On peut imbriquer les balises à l'infini, il faut juste penser à bien fermer toutes les balises dans l'ordre
  • Chacune des balises HTML à cependant des restrictions sur les éléments qu'elle peut contenir

Notions de base

L'héritage

On parle d'héritage lorsqu'il y a imbrication (sur un ou plusieurs niveaux) de balises

<div>
    <ul>
        <li>Texte 1</li>
        <li>Texte 2</li>
        <li>Texte 3</li>
    </ul>
</div>
  • <div> est un parent de <ul>
  • <div> est un ancêtre des <li>
  • <ul> est l'enfant de <div>
  • <ul> est le parent des <li>
  • les <li> sont les enfants du <ul>
  • les <li> sont les descendants du <div>

Notions de base

Les balises block (en bloc)

  • <p>, <div>, <h1>, <ul>, ..., sont des balises block
  • Ces balises se comportent comme un rectangle qui prend toute la largeur et la hauteur du texte qu'elles contiennent
  • Elle peuvent imbriquer des balises inline et certaines autres balises block

Notions de base

Les balises inline (en ligne)

  • <a>, <span>, <em>, <strong>, ..., sont des balises inline
  • Ces balises permettent de différencier une partie du texte
  • Elle ne se comportent pas comme des rectangles, elles s'affichent les une à la suite des autres
  • Elle peuvent imbriquer d'autres balises inline mais pas de balises block

La sémantique

La sémantique

Késako ?

  • L'idée générale est que toutes les balises ont un sens
  • Vous avez peut être noté que le nom de la balise renvoie directement à une abbréviation :
    • <p> pour un paragraphe (paragraph)
    • <h1> pour un titre de premier niveau (heading 1)
    • <h2> pour un titre de deuxième niveau (heading 2)
    • <ul> pour une liste non ordonnée (unordered list)
    • <a> pour un lien (anchor)
    • etc.
  • Le but du jeu est donc d'utiliser la bonne balise, au bon endroit, au bon moment

La sémantique

Exceptions

  • Deux balises sont utilisées en HTML mais n'ont pas d'intérêt sémantique (de sens) :
    • la balise <div> (block) pour créer une division et regrouper des blocs entre eux
    • la balise <span> (inline) pour différencier et agrémenter des mots
  • Ces balises donnent du style mais pas du sens

La sémantique

Quel est l'intérêt ?

  • Déjà un aspect hautement qualitatif :
    • On emploie les balises pour leur vocation initiale (c'est le W3C qui va être content !)
    • On garantit une compatibilité et un bon rendu sur les navigateurs (meilleure analyse du code)
  • Pour les moteurs de recherche :
    • Google préfère qu'on lui serve une page bien composée pour son indexation
    • Ca ne fait donc pas de mal au référencement de son site dans les moteurs !

La sémantique

Quel est l'intérêt ?

  • Pour l'accessibilité aux personnes handicapées :
    • Celles-ci utilisent généralement une aide à la lecture de page HTML (comme pour les non-voyants par exemple)
    • Les lecteurs et matériels pour handicapés interprètent mieux un code respectant la sémantique
  • Sinon pour l'internaute lambda :
    • Globalement aucune différence !
    • On peut avoir le même rendu visuel de la page avec une mauvaise utilisation des balises qu'en les utilisant correctement

    • La balise racine <html> qui contient toutes les autres

    • La balise du corps de page <body> visible

CSS

Définition

CSS (Cascading Style Sheets) : feuilles de style en cascade

  • Les styles sont utilisés pour définir la présentation des éléments HTML
  • Les feuilles de styles sont stockées dans des fichiers CSS
    • Cela permet de tout centraliser et de gagner du temps

Définition

Le fond et la forme

  • Les styles ont été ajouté en HTML 4.0 pour résoudre des problèmes
  • Le langage HTML n'avait jamais été conçu pour contenir des balises de formattage
  • Uniquement pour décrire le contenu dun document (sémantique)
  • Des balises de présentation comme <font>, <center> ont fait leur apparition en HTML 3.2
  • Le balisage mélangeant fond et form devenait alors un cauchemar pour les développeurs
  • Le W3C a donc décidé de créer le CSS
  • En HTML 4.0 on pouvait totalement séparer le fond de la forme

Notions de base

Notions de base

La syntaxe

  • Une règle CSS à deux partie, un sélecteur et une ou plusieurs déclarations 
    /* Ma première règle CSS */
h1 { color: blue; font-size: 12px; }
  • Le sélecteur est l'élement HTML que vous souhaitez décorer (h1)
  • Chaque déclaration consiste en une propriété (color) et une valeur (blue)
  • Chaque déclaration doit être terminée par un point-virgule (;)
  • La propriété est l'attribut de style que vous souhaitez modifier
  • Chaque propriété doit avoir une valeur
  • Les commentaires commence par /* et finissent par */

Notions de base

Les classes

  • Dans l'exemple précédent tous les titres de niveau 1 (h1) seront stylisés
  • Pour n'en styliser que certains on peut sélectionner des éléments HTML en leur attribuant une ou plusieurs classes CSS 
    <p class="information">Acheter du pain à la boulangerie</p>
  • Plusieurs classes sont séparées par des espace : 
    <p class="information important">Ne pas oublier le beurre</p>
  • Les classes s'écrivent avec un point (.) en CSS : 
    .information { font-weight: bold; }
.important { color: red; }
  • Les classes sont réutilisables et peuvent aussi s'appliquer à d'autres balises (si les propriétés sont reconnues)

Notions de base

Les identifiants

  • Les élements HTML peuvent être identifiés directement (attribut id) 
    <div id="header">...</div>
  • Pour le cibler en CSS il faudra utiliser un dièse (#) 
    #header { border: 1px solid black; }
  • Un identifiant n'est utilisable qu'une seule fois par page

Notions de base

L'appel aux feuilles de style

  • Les feuilles de style externes sont apellées via la balise <link> 
    <link rel="stylesheet" href="layout.css" type="text/css" media="screen" />
<link rel="stylesheet" href="text.css" type="text/css" media="screen" />
<link rel="stylesheet" href="print.css" type="text/css" media="print" />

Notions de base

L'appel aux feuilles de style

  • Les règles CSS peuvent aussi être utilisées en interne dans la page (déconseillé cependant)
  • Il faut pour cela placer les règles dans le <head> et les encadrer par une balise <style> 
    <head>
	<style type="text/css">
		hr {color:sienna;}
		p {margin-left:20px;}
	</style>
</head>

Notions de base

Styles en ligne

  • On peut aussi définir des styles en ligne par élément HTML
  • Cette méthode va en contradiction avec le principe de séparation du fond et de la forme (à utiliser avec précaution donc)
  • Il suffit de mettre les règles CSS dans l'attribut style de l'élément HTML
  •  <p style="color:sienna; margin-left:20px;">Un paragraphe</p>

Notions de base

La priorité des appels

  • En fonction d'où seront définies les règles CSS, les règles de priorité ne sont pas les même (priorité croissante) :
    1. La feuille de style système du navigateur
    2. Les feuilles de styles externes
    3. Les styles internes
    4. Les styles en ligne

Notions de base

Le positionnement des blocs

Notions de base

Le positionnement relatif

  • Par défaut les navigateurs affichent les éléments HTML block dans l'ordre normal du flux (ordre de lecture code HTML)
  • En position relative, un élément peut être décalé verticalement et/ou horizontalement
  • Même si cet élément est décalé l'élement suivant suivra normalement le flux (il se peut du coup qu'il y ait des chevauchements)
  • La propriété CSS à utiliser est position et la valeur relative 
    Une <span style="position: relative; background-color: yellow; top: 5px; left: 10px;">information</span> très importante
  • La position relative doit être suivie des propriétés de placement (top ou bottom, left ou right)

Notions de base

Le positionnement absolu

  • La position absolue permet de placer une boîte par rapport aux limites de son conteneur
  • Le conteneur parent doit avoir la position:relative
  • L'élément positionné en absolu sera totalemen retiré du flux

Notions de base

Le positionnement flottant

  • Le positionnement flottant (float) retire une boîte du flux normal pour la faire flotter à droite ou à gauche
  • Le contenu suivant cette boîte flottante s'écoule le long de celle-ci dans l'espace libre
  • Le navigateur place la boîte jaune puis reprend le flux normal dans l'espace libre à gauche pour afficher la boîte verte

Les fichiers reset

Les fichiers reset

Les fichiers quoi ?

  • Les navigateurs sont livrés avec une CSS système (on l'a vu plus haut)
  • Les feuilles de style systèmes sont légèrement différentes d'un navigateur à l'autre
  • On ne peut pas s'en servir directement, juste la surcharger (écrire des règle CSS par dessus qui les invalident)
  • Certains développeurs ont donc eu l'idée de créer une feuille de style qui va remettre à zéro celle du navigateur
    • Suppression des marges,
    • Des espacements,
    • Alignement des polices,
    • ...
